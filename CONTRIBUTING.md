

# Contributing to ContinuumPort

Thank you for your interest in ContinuumPort! This project is currently in **early development**. We value contributions that are not just technically sound, but philosophically aligned with our core axiom: **continuity of direction, not continuity of being.**

By contributing, you help us build a "Semantic RAW" format that ensures human sovereignty over digital tasks.

All contributions must respect the deliberate exclusions defined in [`docs/boundaries.md`](https://github.com/giorgioroth/ContinuumPort/blob/main/docs/boundaries.md). We operate under a strict **Non-Anthropomorphic Constraint**:

* **We standardize the Task (The Stack).**
* **We never standardize the Persona (The Sleeve).**

For a deep dive into this distinction, see [`docs/THE_STACK_ESSAY.md`](https://github.com/giorgioroth/ContinuumPort/blob/main/docs/THE_STACK_ESSAY.md).

## üî¥ Contributions NOT Accepted (Hard Limits)

To maintain the integrity of the project, we will **reject** any proposal that:

1. **Injects Identity**: Proposals to transport personality, emotional state, or behavioral "vibes."
2. **Simulates Self**: Features that imply memory persistence beyond the immediate task requirements.
3. **Blurs the Sensor/Developer Line**: Any change that allows the AI (Regen Engine) to override the human (Master Sensor) intent.
4. **Anthropomorphic Marketing**: Using language that suggests the AI is "becoming" or "remembering" the user.

*Proposals violating these limits will be closed with a reference to our philosophical boundaries.*
